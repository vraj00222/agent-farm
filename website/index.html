Features

Built for the way you actually work.

Most coding agent tools are built around one agent at a time. Agent Farm assumes you want six.

01

Isolated git worktrees

Each task runs in ../<repo>-<id>/ on a fresh agent/<id> branch. Claude can edit anything it wants — your main checkout never sees a byte of it.

02

Real parallelism

Three agents run at once by default. The fourth queues. Zero file conflicts because zero shared filesystem state — that's what worktrees buy you.

03

Live event view

Watch each agent's tool calls, file reads, edits, bash invocations and assistant messages stream in. Token usage and cost surface live, per agent.

04

Cherry-pick the wins

Diffs render in the right pane. Accept the ones that worked with one click, ditch the ones that went off the rails, keep the worktree around for inspection.

05

Hybrid commit handling

Each prompt asks claude to commit. If it forgets, Agent Farm auto-commits whatever was left. Cherry-pick always works, every time.

06

Bring your own login

No new account, no new bill. Agent Farm shells out to your existing claude CLI — Pro, Max, Team, or API key. Your usage, your billing.

How it works

Type a task. Get N worktrees.

Every prompt becomes a branch, a worktree, and a claude process. Three primitives, end to end.

  1. 1 You type a task — Agent Farm slugifies it into an ID.
  2. 2 git worktree add creates a sibling folder on a fresh agent/<id> branch.
  3. 3 claude spawns inside that folder with --output-format stream-json.
  4. 4 Events stream back — tool calls, edits, bash runs, results. You watch, navigate, review.
  5. 5 When you're happy: git cherry-pick agent/<id>. One line, copy-pasteable.

Also a CLI

Same engine, terminal-native.

Don't want a GUI? npm i -g agent-farm and the same orchestrator runs in your terminal. Perfect for SSH sessions, CI pipelines, and remote dev boxes.

Install via npm → 
# install
$ npm i -g agent-farm

# open the live TUI
$ agent-farm

# pre-load tasks
$ agent-farm \
    "fix the JWT bug" \
    "add a rate limiting test"

# inspect a finished session
$ agent-farm status
$ agent-farm logs fix-jwt-bug
