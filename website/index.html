№ 01 — Manifest

Six tasks.
Six worktrees.
One you.

An open-source orchestrator for Claude Code. Each task runs in an isolated git worktree, in parallel, with the agents’ work surfaced as reviewable diffs.

Sometime around the third hour of pair-programming with Claude Code on a single problem, you start to wonder if you’re doing this right. The agent is fast. You are not. It finishes a file in a minute and waits while you read it. You push to a branch, switch context to the next task, and the agent waits some more.

What if it didn’t have to.

Agent Farm is a small, open-source orchestrator. It spawns N Claude Code processes at once, one per task. Each runs inside an isolated git worktree — a sibling folder of your repository on its own branch. They never see each other’s edits. You see all of them.

Mechanism

A worktree, in case it’s been a while: git worktree add creates a second working directory pointing at the same object store as your main checkout, on a fresh branch. Two worktrees on one repo is normal. Six is unusual but not strange. The cost is disk — for a fifty-megabyte repo, six worktrees run you about three hundred megabytes. Acceptable.

Agent Farm automates the lifecycle. You type a task; it slugifies an id, creates the worktree, spawns claude inside it, and starts streaming events back. Up to three sessions run in parallel by default. The fourth queues automatically. 

$ agent-farm

type & enter   spawn a new task
↑ / ↓          select session
tab            tail / diff
ctrl+c         quit

The Fan-Out

Inside each worktree, claude runs with --output-format stream-json --verbose. Tool calls, file reads, edits, bash invocations and assistant messages stream back as a structured event tail. You watch what each one is doing, navigate between them, switch to the diff view when one finishes.

When the work is done you cherry-pick the wins. The misses look like commits you don’t take. Your main branch stays untouched until you choose to touch it. 

$ git cherry-pick agent/fix-jwt-expiry-bug
$ git cherry-pick agent/add-rate-limit-test
$ git worktree remove ../repo-migrate-zod-v4 --force
                                      (that one went off the rails)

What it doesn’t do

It doesn’t host anything. There is no server in the data path. It doesn’t store your prompts. It doesn’t have a freemium tier or a signup screen. It doesn’t replace the claude CLI — it shells out to the one already installed on your machine, signed into whichever plan you already pay for. Pro, Max, Team, API key. Your usage, your billing.

It also doesn’t pretend to be a fancier model than it is. The cost of running ten parallel agents is roughly ten agents’ worth of tokens. There’s no magic compression or shared context. Each agent is independent, on purpose, because that is what makes the conflict-free guarantee work.

Install

The CLI works in any terminal:

$ npm i -g agent-farm
$ agent-farm

The macOS app ships the same orchestrator with a native UI.
